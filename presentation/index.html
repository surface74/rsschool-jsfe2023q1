HTTP authentication
  1. The server, when an unauthorized client accesses a protected resource, sends the HTTP status "401 Unauthorized" and adds the "WWW-Authenticate" header indicating the authentication scheme and parameters.
  2. The browser, upon receiving such a response, automatically displays the username and password input dialog. The user enters their account details.
  3. In all subsequent requests to this website, the browser automatically adds the HTTP header “Authorization”, which transmits user data for authentication by the server.
  4. The server authenticates the user according to the data from this header. The decision to grant access (authorization) is made separately based on the user's role, ACL, or other account data.
